A. Status Summary + List

Top: Count cards (items below threshold, flagged). Below: Simple list of items needing action, sorted by priority.

Pros

  • Quick visual: "apa yang butuh action"
  • Simple, mobile-friendly
  • Focus on priority items only

Cons

  • Don't see ALL items (only at-risk)
  • Need separate view for full inventory

B. Simple Flat List

All items in one list: Item name, Current qty, Threshold, Status (color-coded). Sortable by status (red first) or name.

Pros

  • See complete inventory at once
  • Easy to scan
  • No hidden items
  • Familiar (like spreadsheet)

Cons

  • May be long list if many items
  • Less visual priority

C. Kanban Columns

Group by status: Below Threshold | OK | Flagged. Each column shows items count and list. Visual grouping for quick scanning.

Pros

  • Visual grouping by status
  • Quick "what needs action" scan
  • Scalable (works with many items)
  • Touch-friendly (cards)

Cons

  • May not show ALL items compactly

💡 Context: Crew monitoring is for daily operational checks — "am I ready for opname?", "stok X berapa?", "ada item rusak?". Different from SPV's multi-outlet alert aggregation.
